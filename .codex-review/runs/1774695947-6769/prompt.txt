Claim: "79 services mixing business logic + data access → need Repository layer to abstract data access"
Impact: New pattern across entire backend

### REC-2: DTOs / Value Objects (High)
Claim: "Besides API Resources (output), need Input DTOs. P2 should be renamed DTO Layer."
Impact: Changes P2 scope significantly

### REC-3: Action / UseCase / Command Pattern (High)
Claim: "Giant services >1500 LOC should be split into Action classes (1 class = 1 responsibility)"
Impact: Overlaps with P4 (Action Classes)

### REC-4: Standardized API Error Handling (High)
Claim: "No custom Exception Handler + ApiErrorResource. Error responses are inconsistent."
Impact: Cross-cutting, could be P1 or P2

### REC-5: Rate Limiting & Throttling (Medium-High)
Claim: "Only permission middleware, no API abuse protection"
Impact: Security concern

### REC-6: API Documentation (Scribe/OpenAPI) (Medium)
Claim: "100+ endpoints undocumented, no auto-generated docs"
Impact: Developer experience

### REC-7: Laravel Horizon + Telescope (Medium)
Claim: "Queue has 2 jobs but no monitoring dashboard"
Impact: Operations

### REC-8: Contracts & Interfaces (Medium)
Claim: "Tight coupling Service→Model. Should inject via Interface."
Impact: Architecture purity

### REC-9: Database Indexing & Query Optimization (Low-Medium)
Claim: "87 DB::raw calls, no index audit or slow query log analysis"
Impact: Performance

### REC-10: Feature Flags (Low)
Claim: "No way to toggle features without deploy"
Impact: Deployment safety
