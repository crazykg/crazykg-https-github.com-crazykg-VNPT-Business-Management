## Import Instructions

1. The customer data can be imported directly into the `customers` table in the database
2. The support service group data (from Excel) can be imported into the `support_service_groups` table
3. The `customer_id` in the support service groups should correspond to the `id` from the customer data
4. Use the API endpoint at `http://localhost:5174/api/v5/support-service-groups` for programmatic imports
5. Alternatively, use the bulk import endpoint at `http://localhost:5174/api/v5/support-service-groups/bulk`

## Field Descriptions

- `id`: Unique identifier for the customer
- `customer_code`: Unique code for the customer
- `customer_name`: Full name of the customer
- `address`: Physical address of the customer
- `customer_sector`: Industry sector of the customer (e.g., HEALTHCARE)
- `group_name`: Name of the support group
- `group_code`: Unique code for the support group
- `is_active`: Whether the customer/group is active (1 for active, 0 for inactive)

## Data Validation Rules

- All customer codes must be unique
- Customer names should not be empty
- Group names within each customer must be unique
- Customer IDs in support groups must exist in the customers table
