UPSELL RECOMMENDATION PIPELINE (CustomerInsightService):
═════════════════════════════════════════════════════════════════════════════════════════

INPUT: customer_id
  │
  ├──────► Step 1: Get products customer ALREADY USES
  │         └─► Query: contract_items → contracts (customer_id=?) → products
  │
  ├──────► Step 2: Check for HIS products
  │         └─► If customer has ANY HIS → Exclude ALL HIS from suggestions
  │
  ├──────► Step 3: Pre-aggregate product POPULARITY (single query)
  │         └─► For each product: COUNT(DISTINCT customer_id) where using this product
  │         └─► This is social proof: "47 other customers use this"
  │
  ├──────► Step 4: Query UPSELL CANDIDATES
  │         └─► active products NOT in customer's contracts
  │         └─► Order by: GROUP_A first, then popularity DESC
  │         └─► Limit: 8 suggestions
  │
  ├──────► Step 5: Get REFERENCE CUSTOMERS (≤3 per product)
  │         └─► Who else is using each suggested product?
  │         └─► Provides social proof
  │
  └──────► OUTPUT: upsell_candidates[] with:
           - product_id, product_name, standard_price
           - service_group (A/B/C), is_priority (A=true)
           - popularity (count of other customers)
           - reference_customers (≤3 names as proof)

CUSTOMER INSIGHT ENDPOINT: GET /api/v5/customers/{id}/insight
═════════════════════════════════════════════════════════════════════════════════════════

Response Structure:
{
  data: {
    customer: { id, name, sector, facility_type, bed_capacity, ... },
    
    contracts_summary: {
      total_count: 12,
      total_value: 500000,
      active_value: 450000,
      by_status: { SIGNED: 8, RENEWED: 3, DRAFT: 1 }
    },
    
    services_used: [                      ← Products already in contracts
      {
        product_id: 5,
        product_name: "HIS Pro",
        contract_count: 2,               ← 2 contracts use this
        total_value: 100000,
        service_group: "GROUP_A"
      },
      ...
    ],
    
    opportunities_summary: {              ← Open sales pipeline
      total_count: 3,
      total_amount: 150000,
      by_stage: { NEW: 1, QUOTED: 2 }
    },
    
    crc_summary: {                        ← Support cases
      total_cases: 15,
      open_cases: 3,
      by_status: { "analysis": 2, "in_progress": 1 }
    },
    
    upsell_candidates: [                  ← RECOMMENDATIONS ⭐
      {
        product_id: 8,
        product_name: "ERP Module",
        standard_price: 25000,
        unit: "per year",
        service_group: "GROUP_B",
        service_group_label: "Dịch vụ nhóm B",
        reason: "Dịch vụ nhóm B",
        popularity: 42,                   ← 42 customers use this
        is_priority: false,
        reference_customers: ["Hospital X", "Hospital Y", "Clinic Z"]
      },
      ...
    ]
  }
}

Cache: 5 minutes (v5:customer-insight:{id}:v1)
Invalidated on: customer/contract/CRC updates

PERFORMANCE OPTIMIZATIONS:
═════════════════════════════════════════════════════════════════════════════════════════

1. VAT Rate Memoization
   • Load business_domains once per request
   • Use PHP map instead of SQL CASE per product

2. Pre-aggregated Popularity
   • Single derived table query (not correlated subqueries)
   • Avoids N queries for N products

3. Covering Indexes
   • idx_ci_product_contract on contract_items
   • idx_ps_contract_expected_status on payment_schedules

4. Query Caching
   • 5-minute cache on insight per customer
   • Invalidated on data changes
